## 🧠 핵심 정리

* 생성자 함수는 `new`와 함께 호출되며
  → 객체 생성, this 바인딩, 객체 반환까지 자동 수행

* `new` 호출 시 내부 동작
  → 빈 객체 생성 → this 바인딩 → 속성 할당 → 객체 반환

* `new` 없이 호출 시 this가 전역 객체를 가리켜
  → **전역 오염 발생 가능**

* `new.target`을 활용하여
  → 생성자 함수의 **안전한 호출 보장 가능**

---

* 자바스크립트는 **프로토타입 기반 객체지향 언어**

* 모든 객체는 `[[Prototype]]`을 가지며
  → 상위 객체와 연결된 **상속 구조 형성**

* 프로퍼티 동작 방식

  * 조회 → 프로토타입 체인을 따라 상위 탐색
  * 할당 → 현재 객체에 생성 (상위 객체의 프로퍼티를 가림)

* 프로토타입 메서드를 호출하더라도
  → `this`는 항상 **호출한 객체 기준으로 바인딩**

---

* 배열 메서드 분류

  * 원본 변경 (Mutating)
    → `push`, `pop`, `shift`, `unshift`, `splice`, `reverse`, `sort`

  * 새로운 배열 반환 (Non-mutating)
    → `slice`, `map`, `filter`

* 고차 함수 역할

  * `map` → 데이터 변환
  * `filter` → 조건 기반 추출
  * `reduce` → 누적 연산
  * `find` → 첫 번째 요소 반환
  * `some` / `every` → 조건 검증 (boolean)

* `sort()`는 기본적으로 문자열 기준 정렬
  → 숫자 정렬 시 compare 함수 필요

---

* 표준 내장 객체

  * String → 검색(`includes`, `indexOf`), 추출(`slice`), 분할(`split`), 공백 제거(`trim`)
  * Math → 난수 생성(`random`), 정수화(`floor`), 반올림(`round`, `ceil`)
  * Date → 현재 시간 처리 (`toLocaleString`, `getFullYear`, `getMonth` (0부터 시작))
