## 🧠 핵심 정리

* 함수 재생성과 `useCallback`

  * 컴포넌트 렌더링 시 함수는 매번 새로 생성됨
  * `useCallback`은 의존성 값이 변경될 때만 함수 재생성
  * 자식 컴포넌트에 함수 props 전달 시 **불필요한 리렌더링 방지**

* 함수 의존성과 `useEffect`

  * 함수도 참조값이기 때문에 의존성 배열 관리 필요
  * `useCallback` 없이 전달 시, 매 렌더링마다 새로운 함수로 인식되어 `useEffect` 반복 실행 발생

* 상태 관리 방식 비교

  * `useState`: 값 유지 + 변경 시 리렌더링 발생
  * 일반 변수: 렌더링 시 초기화되어 값 유지 불가
  * `useRef`: 값 유지 가능하지만 리렌더링 발생하지 않음

* `useRef`의 역할

  * `.current`를 통해 값 저장
  * 렌더링 없이 값 유지
  * DOM 요소 직접 접근 가능

* DOM 제어 (포커스 처리)

  * `useRef`를 활용하여 특정 input 요소에 자동 포커스 적용
  * 마운트 시점(`useEffect`)과 결합하여 UI 제어 가능

* Custom Hook을 통한 로직 분리

  * 반복되는 로직을 Hook으로 추출하여 재사용
  * 컴포넌트 구조 단순화 및 가독성 향상

* 비동기 처리 방식의 발전

  * Callback: 중첩 구조로 가독성 저하 (Callback Hell)
  * Promise: `.then` 체이닝으로 흐름 개선
  * async/await: 동기 코드처럼 작성 가능하여 가독성 최적화

## ⚙️ 적용

* `useCallback`을 활용하여

  * 상태 변경 시에만 함수가 재생성되도록 제어
  * 자식 컴포넌트의 불필요한 `useEffect` 실행 방지

* 부모 → 자식 함수 전달 구조에서

  * 함수 참조값 변화로 인한 렌더링 문제 직접 확인 및 개선

* `useState`, 변수, `useRef`를 비교 테스트하여

  * 렌더링 여부 및 값 유지 차이 명확히 확인

* `useRef`를 활용하여

  * 로그인 입력창 자동 포커스 기능 구현

* Custom Hook(`useDelayTimer`) 구현을 통해

  * 타이머 기반 상태 변경 로직 분리 및 재사용 구조 설계

* Callback → Promise → async/await로 동일 로직을 변환하며

  * 비동기 흐름 개선 과정 직접 비교
