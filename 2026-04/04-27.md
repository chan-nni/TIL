## 🧠 핵심 정리

* 클라이언트 상태 vs 서버 상태

  * 클라이언트 상태: UI 및 사용자 인터랙션 상태 (Zustand)
  * 서버 상태: DB에 저장된 데이터 (React Query)
  * 상태를 분리하여 관리하면 유지보수성과 구조가 명확해짐

* Zustand와 `persist`

  * 전역 상태를 관리하고 여러 컴포넌트에서 공유 가능
  * `persist`를 사용하여 localStorage에 상태 저장
  * 새로고침 이후에도 상태 유지 가능

* React Query의 역할

  * 서버 데이터를 “저장”하는 것이 아니라 요청과 상태를 관리
  * `useMutation`: 데이터 생성/수정/삭제 요청
  * `invalidateQueries`: 데이터 변경 이후 최신 데이터 재요청

* React Hook 실행 순서

  * Hook은 항상 동일한 순서로 호출되어야 함
  * 조건문 내부에서 호출 시 에러 발생
  * 모든 Hook을 최상단에서 실행하도록 구조 수정

## ⚙️ 적용

* Zustand 활용

  * 로그인 상태 및 UI 상태를 전역으로 관리
  * `persist`로 localStorage와 동기화

* React Query 활용

  * 리뷰 등록 기능 구현
  * `invalidateQueries`로 최신 데이터 자동 반영

* Hook 실행 순서 문제 해결

  * 조건부 렌더링 위치 변경으로 에러 해결
