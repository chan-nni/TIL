## 🧠 핵심 정리

* DOM 요소 선택과 다중 요소 컬렉션
  * `querySelectorAll`은 상태가 고정된 **정적 컬렉션(NodeList) 반환**
  * `getElementsByClassName` 등은 DOM 변화가 실시간 반영되는 **동적 컬렉션(HTMLCollection) 반환**

* 동적 컬렉션(`HTMLCollection`) 순회 주의점
  * 반복문 중 요소 제거 시 인덱스 꼬임 등 **요소 누락 버그 발생**
  * 배열로 변환하거나 정적 컬렉션을 사용하여 **안정적인 제어 보장**

* DOM 탐색 시 텍스트 노드 포함 여부
  * `children` 등은 요소만 반환하여 **안전한 탐색 가능**
  * `childNodes`는 공백, 줄바꿈 등 텍스트 노드까지 포함하여 **요소 기준으로 탐색할 경우 예상과 다른 결과가 발생**할 수 있음

* 텍스트 조작 시 `textContent` 활용
  * 요소에 직접 접근하여 **내부의 모든 텍스트 반환**
  * 문자열 할당 시 기존 자식을 모두 제거하고 **안전하게 텍스트로 교체**

* DOM 조작 및 렌더링 방식
  * `innerHTML`, `insertAdjacentHTML`은 간편하지만 사용자 입력을 검증 없이 그대로 삽입하는 경우 **XSS 공격에 취약**
  * `createElement` + `appendChild` 조합이 보안 및 유지보수 측면에서 **가장 안전한 정석**

* 성능을 고려한 DOM 렌더링 최적화
  * 여러 요소를 메모리에서 먼저 조립하는 `DocumentFragment` 활용
  * DOM 트리에 한 번에 반영하여 **불필요한 리플로우(Reflow) 발생 최소화**

* 어트리뷰트(Attribute) vs 프로퍼티(Property)
  * Attribute는 HTML에 정의된 **초기(정적) 상태** (`getAttribute`)
  * Property는 DOM 객체로 파싱된 후의 **현재(동적) 상태** (`.value`)
  * 사용자 입력 등 지속 변화하는 상태는 **반드시 Property 사용**

* 데이터 바인딩과 상태 제어
  * `data-*`와 `dataset`을 통해 DOM 요소에 **비즈니스 데이터 직접 연결**
  * 인라인 스타일 직접 조작을 피하고 `classList`를 활용하여 **JS와 CSS의 관심사 분리**

## ⚙️ 적용

* `HTMLCollection` 순회 중 클래스 제거 동작 테스트 및
  * 실시간 반영 특성으로 인한 **요소 누락 버그 직접 확인**

* `querySelectorAll` 및 배열 변환을 활용하여
  * **안정적인 다중 요소 제어 로직 구현**

* `createElement`, `appendChild`, `DocumentFragment`를 조합하여
  * 렌더링 성능(Reflow)을 개선한 사용자 입력 기반 **동적 리스트 UI 구현**

* `dataset`과 이벤트 객체(`event.target`)를 연동하여
  * 클릭된 요소의 **고유 데이터 식별 로직 처리**

* `classList.toggle`을 활용하여
  * JS와 CSS의 관심사를 분리한 **UI 상태 제어 적용**
